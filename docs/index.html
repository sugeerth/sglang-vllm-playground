⚡

SGLang — RadixAttention

Uses a radix tree to cache and reuse KV-cache across requests sharing common prefixes. Excels at structured generation and multi-turn workloads with high prefix overlap.

📚

vLLM — PagedAttention

Manages KV-cache in non-contiguous memory pages (like OS virtual memory). Eliminates fragmentation and enables near-zero waste memory allocation for large batches.

📈

Side-by-Side Comparison

Send identical prompts to both engines simultaneously. Compare latency, throughput (tokens/sec), time-to-first-token, and output quality in one view.

📊

Batch Benchmarks

Run N prompts through both backends and get aggregate statistics: avg/P50/P99 latency and throughput, with interactive Chart.js visualizations.

🔌

OpenAI-Compatible API

Both SGLang and vLLM expose an OpenAI-compatible /v1/chat/completions endpoint, so the playground works with either engine out of the box.

🎨

Demo Mode

No GPU? No problem. The demo mode simulates realistic inference latency profiles so you can explore the full UI and benchmark workflow without any hardware.

Quick Start
1. Clone & install
git clone https://github.com/sugeerth/sglang-vllm-playground && cd sglang-vllm-playground && pip install -r requirements.txt

2. Run in demo mode (no GPU needed)
python3 app.py --demo

3. Or connect real backends
SGLANG_URL=http://localhost:30000 VLLM_URL=http://localhost:8000 python3 app.py
